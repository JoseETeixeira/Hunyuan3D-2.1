Root cause

The bake is cosine-gated — oblique normals never get written

The 75° gate

MeshRender.py:366 sets bake_angle_thres = 75. In back_project (:1192, :1245) any texel whose surface normal is >75° off the camera axis gets cos = 0 → that view does not write it.

A texel is coloured only if some baked view sees it within 75°. The red-circled slanted ramp/facade walls graze every fixed camera → never written.

Inpaint then smears it

Unwritten texels fall to texture_inpaint (MeshRender.py:1411, cv2.INPAINT_NS, island-blind) which floods them with the nearest neighbour colour.

Result = the "unpainted / wrong colour" oblique patches. The gap is purely camera coverage: the current view sets leave a cone of surface normals unhit.

Coverage signal ready

Algo candidate 1 — UV-normal clustering

Interpolate vertex normals into UV space → per-texel world normal. Cluster gap normals on the sphere; aim a camera down each cluster mean. Direct, needs careful UV-normal extraction + cluster tuning.

Algo candidate 2 — greedy camera set-cover

Sample a Fibonacci-sphere of candidate cameras; greedily pick the top-K that newly cover the most gap texels until a round adds < ε. Reuses render/back_project, no UV-normal math, naturally capped. likely simpler

Resolved decisions (grill-me)

Locked direction

Approach
Auto-detect coverage gaps and auto-place fill cameras. No manual aiming. Same mechanism in both modes.
Activation ⚠
Default ON in both reface and hyface — this CHANGES current behavior of both modes (architecture change, user-validated). Sane env-tunable camera cap kept as a guardrail (uncapped rejected).
Gap =
Zero-coverage texels (cos_map==0) + small dilation border so new paint blends over the inpaint seam. Grazing/low-cos NOT in scope.
Reference
reface → render_textured_view → restyle_to_references → worker.reface (multi-pass, one per gap cluster). hyface → gpt-synth from render_geometry_at + adjacent faces, degrade gracefully without an API key.

Change surface & risks

Touch (additive)

  • webapp/server.py — gap stage in _run_reface + _run_hyface; env cap; /api/reface params if needed.
  • webapp/pipeline.py — coverage/gap-mask + camera-placement helper (back-ends already angle-generic).
  • index.html / app.js — minimal (default-on); optional cap/toggle surface.

Risks

  • Robust gap→camera placement is the crux (decide algo in Design).
  • Cost: each fill view = a diffusion pass (hyface) or gpt call + bake (reface) — bounded by the cap.
  • reface becomes multi-pass; ensure passes composite cleanly over the base.
  • Truly-unseeable cavities still fall to inpaint — coverage shrinks the gap, not always zero.
